2. Arduino y Arquitecturas de Microcontroladores

Arduino es una plataforma open-source basada en microcontroladores, popular por su facilidad de uso en proyectos de electrónica y IoT.

Placas y Microcontroladores Comunes
Arduino UNO
MCU: ATmega328P.
Arquitectura: Harvard.
Arduino MEGA
MCU: ATmega2560.
Características: Más I/O, más memoria.
ESP32 (No Arduino)
MCU: Tensilica LX6.
Características: Wi-Fi y Bluetooth nativos.
Clasificación de Pines y Funciones Esenciales

Los pines del Arduino tienen múltiples funciones. Las capacidades especiales se asocian a números de pin específicos:

Comunicación Serial (UART)

RX - Pin 0: Recepción de datos seriales.

TX - Pin 1: Transmisión de datos seriales.

Comunicación I2C (Inter-Integrated Circuit)

SDA - Pin A4 (Digital 18): Línea de Datos Seriales.

SCL - Pin A5 (Digital 19): Línea de Reloj Serial (Clock).

Comunicación SPI (Serial Peripheral Interface)

SS - Pin 10: Chip Select (Selección de Esclavo).

MOSI - Pin 11: Master Out, Slave In (Envío del Maestro).

MISO - Pin 12: Master In, Slave Out (Recepción del Maestro).

SCK - Pin 13: Serial Clock (Reloj de Comunicación).

Salida Digital (GPIO)

Pines 0 al 19: Pueden configurarse como Entrada (Input) o Salida (Output).

Salida PWM (Pulse Width Modulation)

Pines 3, 5, 6, 9, 10, 11: Permiten simular salidas analógicas (brillo, velocidad de motor).

Entrada Analógica

Pines A0 a A5: Usados para leer señales continuas del mundo real (ej. sensores de temperatura).

Interrupciones Externas

INT0 - Pin 2: Detiene el proceso principal para ejecutar una función urgente.

INT1 - Pin 3: Segundo pin dedicado a interrupciones rápidas.

Hardware Adicional

Botón de Reset: Reinicia el programa desde el inicio.

Shields: Placas modulares que amplían hardware y capacidades.

La Arquitectura Clave: Harvard

Los microcontroladores de Arduino (como el ATmega328P) utilizan la arquitectura Harvard para garantizar un alto rendimiento.

Característica Von Neumann (PC, Servidores) Harvard (Arduino, DSP)
Memoria Unificada: Una sola memoria para Datos e Instrucciones. Separada: Memorias separadas para Datos (SRAM) e Instrucciones (Flash).
Buses Bus Único: Uno para transferir todo. Causa el "Cuello de Botella". Buses Múltiples: Buses independientes para cada memoria.
Ejecución Secuencial: Solo puede buscar una instrucción O un dato a la vez. Paralela: Puede buscar Instrucción y Dato simultáneamente (más rápido).
Aplicación Sistemas de Propósito General. Sistemas Embedded (Microcontroladores, DSP, Arduino).
Usos y Compatibilidad de Arduino
  • Usos: Domótica, robótica educativa, prototipado rápido y monitoreo en IIoT.
  • Compatibilidad: Se conecta a una amplia gama de Sensores (digitales, analógicos) y Actuadores (relés, motores) a través de sus pines de E/S.
  • Aplicaciones Industriales (IIoT): Utilizado para crear módulos de adquisición de datos (DAQs) de bajo costo o sistemas de control sencillos.

3. Arquitectura Híbrida: Nintendo 3DS vs. Von Neumann

La Nintendo 3DS utiliza una arquitectura altamente paralela y especializada para manejar gráficos 3D y retrocompatibilidad, alejándose del modelo simple de Von Neumann.

CPU y Paralelismo
  • CPU Principal: Dual Core ARM11 (268 MHz) para el juego y SO de 3DS.
  • CPU Secundario: Single Core ARM9 para asegurar la retrocompatibilidad con los juegos de Nintendo DS.
  • GPU: DMP Pica200 dedicada, la cual debe renderizar dos frames (400x240) simultáneamente para el efecto 3D estereoscópico.
  • Memoria (FCRAM/VRAM): Tiene 128 MB de RAM principal y 6 MB de VRAM dedicada a la GPU, crucial para el rendimiento gráfico.
Comparación con Von Neumann (Limitación)
Aspecto Modelo Von Neumann (PC Básico) Arquitectura 3DS (Híbrida/Especializada)
Procesamiento Centralizado en una CPU. Distribuido: CPU (lógica), GPU (gráficos), ARM9 (compatibilidad).
Memoria Gráfica VRAM usualmente compartida con la RAM principal (bus único). VRAM Dedicada (6MB): Acceso rápido y directo por la GPU.
Diferencia Clave Limitada por el Cuello de Botella del bus único. Utiliza múltiples núcleos y memorias separadas (característica Harvard) para evitar el cuello de botella en los componentes críticos de juego.
